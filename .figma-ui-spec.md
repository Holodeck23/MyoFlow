## Complete UI Generation Prompt

Build a full UI for MyoFlow (Austrian Therapy Practice Management)

### Project Overview
Design a responsive, production-grade UI for MyoFlow, a single-tenant SaaS for Austrian massage therapists and small clinics. The UI must be bilingual (de-AT / en-AT), privacy-first (GDPR), and compliant-aware for Austrian invoicing and cash register requirements. Output modern, clean screens with a clear information hierarchy and print-ready invoice layouts.

### Target Users & Primary Flows
• Solo Therapist (default persona): sets up services, opens booking, tracks sessions, writes notes, invoices, monitors small-business VAT threshold (Kleinunternehmerregelung).
• Clinic Admin (multi-practitioner): manages practitioners, rooms, schedules, permissions, payouts, reports.
• Front Desk (limited role): calendar, client intake, appointment edits, check-in/out, payments.

### Design System (Austrian Medical)
• Colors
  - Primary deep blue: #1565C0
  - Accent Austrian red: #C8102E
  - Backgrounds: page #F7F7F8, card #FFFFFF, soft section #F0F2F5
  - Text: primary #1F2937, secondary #4B5563, disabled #9CA3AF
  - Borders/dividers: #E5E7EB
  - Status: success #16A34A, warning #F59E0B, danger #DC2626, info #0EA5E9

• Typography
  - UI font: Inter (or system-equivalent)
  - Headings: H1 28/36, H2 22/30, H3 18/28 (semibold)
  - Body: 14/22 (regular), 16/24 for forms (medium)
  - Caption/Meta: 12/18 (regular)

• Layout & Spacing
  - Grid: max-width 1280px, 12-column, 24px gutters
  - Spacing: 8/12/16/24/32/40/64 scale (default 16)
  - Cards: 12px radius, subtle shadows (hover elevation), 1px border

• Components
  - Buttons: primary (blue), secondary (gray), danger, ghost
  - Inputs: large, 44px height; clear error/help states
  - Tabs, segmented controls, breadcrumbs
  - Toasts, modals, drawers
  - Data tables (sticky header, row selection, inline filters)
  - Calendars (month/week/day), kanban, timeline
  - Progress bars, badges, chips, empty states

• Accessibility
  - AA contrast, 44px targets, visible focus, keyboard navigable
  - Date/time formats: DD.MM.YYYY, 24-hour time, € currency

### Global Requirements
• Language switcher: de-AT / en-AT; all key strings translatable (show sample copy in both)
• Role-based nav: Solo Therapist, Clinic Admin, Front Desk
• Audit & GDPR: access logs, consent tracking, data export/delete flows
• Configurable compliance: VAT logic & thresholds are configurable (default 35,000 € for small-business VAT exemption; allow overrides)
• Responsive: desktop first, excellent tablet/mobile
• Branding: upload logo, brand color, booking page theme
• Dark mode: optional

### Primary Navigation (left sidebar)
Dashboard · Calendar · Clients · Sessions/Notes · Invoices & Payments · Products/Packages (optional) · Reports · Messaging · Booking Page · Settings (Business, Team, Services, Compliance, Integrations)

### Technical Implementation Notes
- Use Next.js 14 (App Router) + TypeScript + TailwindCSS
- shadcn/ui components for consistency
- i18n: file-based JSON (/i18n/en.json, /i18n/de.json)
- Zustand for light state management
- Zero external APIs for initial implementation
- Accessibility: AA contrast, keyboard nav, 44px targets

## Kleinunternehmer Widget (Critical Component)
Progress bar showing YTD net revenue vs configurable threshold
States: Normal (<80%), Warning (≥80%), Critical (≥95%)
Text examples:
- EN: "YTD €27,400 of €35,000 threshold (78%). Forecast: threshold in 6.2 weeks."
- DE: "Bisher €27.400 von €35.000-Grenze (78 %). Prognose: in 6,2 Wochen erreicht."
Buttons: "Configure threshold", "View report", "What happens if I exceed?"
